NEP API

  • This tutorial explains how to communicate nodes using the patterns supported in NEP

Client-Server

In the Client-Server pattern a node (Client) sends a request for some data to other node (Server) and then waits for the response to this request. An example of how to create a simple client-server application is shown bellow:

Creates a client in Python

The next Python script shown a simple example of a client node

import nep 
import time 

client = nep.client('127.0.0.1', 8010) # Create a new client instance

while True:
    msg = {"message":"client request"} # Message to send as request
    client.send_info(msg)   # Send request
    print (client.listen_info()) # Wait for server response
    time.sleep(1) # Wait one second

In this example a new instance of the client class is created with the line:

client = nep.client('127.0.0.1', 8010)

For this, the IP and PORT parameters are defined as '127.0.0.1' and 8010 respectivelly.

The function send_info can be used to send a request to the server. This request must be defined as a python dictionary. In this case the dictionary {"message":"hello server"} is defined to be send as request.

The function listen_info waits until the server send its response. This function also reads the message sent by the server.

The client class uses ZeroMQ by deafult as transport layer. In order to communicate with a device that uses a normal python TCP/IP POSIX sockets, the optional parameter transport can be defined as "normal" in the constructor of a client object. For example:

client = nep.client('127.0.0.1', 8010, transport = "normal")

Creates a server in Python

The next Python script shown a simple example of a server node.

import nep  # Import nep library
server = nep.server('127.0.0.1', 8010) #Create a new server instance

while True:
    msg = {"message":"server response"} # Message to send as response
    request = server.listen_info() # Wait for client request
    print (request) 
    server.send_info(msg) # Send server response

In this example a new instance of the server class is created with the line:

server = nep.server('127.0.0.1', 8010)

We need to define the same IP and PORT values that those defined in the client node. The functions listen_info and send_infocan be used to read a client request and send a response respectively. In this example the response to be sent in defined as {"message":"hello client"}.

The function send_info can be used to send a request to the server. This request must be defined as a python dictionary. In this case the dictionary {"message":"hello server"} is defined to be send.

The function listen_info waits until the server send its response. This function also reads the message sent in that response.

We can also uses normal TCP/IP POSIX sockets defining the optional parameter transport as "normal", as shown in the next line:

server = nep.server('127.0.0.1', 8010, transport = "normal") #Create a new client instance

About normal TCP/IP POSIX sockets

In the current version of NEP, messages in normal TCP/IP POSIX sockets must be strings, instead of dictionaries.

About ZeroMQ and normal TCP/IP POSIX sockets

It is nor possible to communicate a ZeroMQ socket and a normal TCP/IP POSIX socket. Therefore you need to be sure that both client and server has the same trasport layer (ZeroMQ or normal TCP/IP POSIX sockets).

Publish-Subscribe

In a Publish-Subscribe pattern, a producer of messages (publisher) send messages without knowing if any receiver (subscriber) exists. An example of how to create a simple publish-subscriber application is shown bellow:

A simple publish-subscribe example

The next Python script can be used to create a new publisher instance.

import nep
import time

node = nep.node("pub")
conf = node.conf_pub(network = "direct", port= 7075)
pub = node.new_pub("test", conf)
while True:
    msg = {"message":"hello world"}
    pub.send_info(msg)
    time.sleep(1)

The next Python script can be used to create a new subscribe instance.

import nep

node = nep.node("sub")
conf = node.conf_sub(network = "direct", port= 7075)
sub = node.new_sub("test", conf)
while True:
    s, msg = sub.listen_info()
    if s:
        print (msg)

Create a new node

In order to perform the communication between of two or more nodes using the Publish-Subscribe pattern, we need to import the nep library and create a new instance of the node class. The constructor takes one main parameter and an optional parameter. The main parameter defines the name of the node. This name must to be different for each node in the network. In the above examples we define the name of the nodes as publisher_node and subscriber_node for the publisher and subscriber respectivelly. When the optional parameter useROS is set to True a ROS node will be created (Only for Linux-based desktop machines).

node = nep.node("publisher_node" , useROS = True)

Define the publisher and subscriber configuration

Direct network

The NEP library suports direct endpoints conections (only ZeroMQ and nanomsg) and P2P master service conections (in ZeroMQ, nanomsg and ROS). The pub_conf sub_conf methods in the node class allow to define the configuration of the network to be used in the publisher and subscriber respectively.

In order to define a direct endpoints conection in a local network (127.0.0.1) we can use the next line for the publisher:

conf = node.conf_sub(network = "direct", port = "9090")

The subscriber configuration able to communicate with the publisher defined above is:

conf = node.conf_pub(network = "direct", port = "9090")

In ZeroMQ a publish-subscribe pattern can have 3 different modes, one2many, many2one and many2many. In a one2many mode only one publisher can be connected to a port to publish data. In this mode multiple susbcribers can read the data send by this publisher.In a many2one mode multiple publisher can send data in the same port. However only one subscriber can read the data sent by those publishers. Finally, in a many2many multiple publishers can send data that can be readed by multiple subscribers. The mode used for communication can be defined in the mode parameter. In NEP a direct network only supports one2many and many2one modes.

P2P network

In NEP a P2P network only supports one2many, many2one and many2many modes. However before launching a node using a P2P configuration a node called NEP Master must be executed. This node is in charge to manage the connections in a P2P network and it should never be closed during the execution of nodes using the P2P configuration.

The next line shows an example of how a many2many publisher can be defined:

conf = node.conf_pub(transport = "ZMQ", network = "P2P", mode = "many2many") # Select the configuration of the publisher

However, due that the default value of the parameter network in "P2P", the definition of this parameter can be omited as:

conf = node.conf_pub(transport = "ZMQ", mode = "many2many") # Select the configuration of the publisher

The next line shows an example of how a many2many subscriber can be defined:

conf = node.conf_sub(transport ="ZMQ", mode = "many2many") # Select the configuration of the subscriber

We can use the transport parameter to use nanomsg instead ZeroMQ. The next line shows how a one2many and P2P nanomsg publisher is defined.

conf = node.conf_pub(transport = "NN")

Create instances for publishers and subcribers

After defining the configuration of the network we can use the new_pub() and new_sub() methods to create a new publisher and subscriber instances respectivelly. This functions takes two main parameters. The first parameter is the topic in which information will be sent. The second parameter is used to set the configuration of the publisher/subscriber instance. An example of definition of a new publisher is show in the next line:

conf = node.conf_pub(transport = "ZMQ", mode = "many2many")
pub = node.new_pub("test", conf)

This publisher will be publish data in the "test" topic, using ZeroMQ in a many2many and P2P network. A subscriber able to read the data send by this publisher can be defined as:

conf = node.conf_sub(transport ="ZMQ", mode = "many2many")
sub = node.new_pub("test", conf)

Send information

In order to send information to the network we can use the methods send_string() or send_info().

When using the send_string() method we can send the information as strings. An example is show bellow:

msg = "hello world"
pub.send_string(msg)

The function `send_info() can be used to send complex messages in JSON format (python dictionaries) using ZeroMQ and nanomsg or ROS geometric messages such as Point, Pose, Quaternion, etc. An example of sending a ZeroMQ message is shown bellow:

msg = {"message":"hello world"}
pub.send_string(msg)

Listen information

In order to get the information that a publisher instance send to the newtwork we can use thelisten_string() and listen_info() functions. On the one hand, the listen_string() function in a subscriber instance is used to read data sent by a publisher with the send_string() function. The information send and read by these function must be strings. On the other hand, the listen_info() function in a subscriber instance is used to read data sent by a publisher with the send_info() function. In this case the information send and read must be dictionaries.

Communication with other PCs

Different computers using different operating systems can be connected using a direct network configuration. For we need to have connected this devices to the same WiFi network and specify the IP address of one of the PCs in the network and the same port. The IP value will depend of the publish-subscribe configuration mode.

Mode IP value of
one2many Publisher
many2one Subscriber
many2many Master Node

For example if a Publisher node in a PC with IP=“192.168.11.1” need to communicate with a Subscriber node in other PC with IP=“192.168.11.2”, the configuration of the publisher can be as the next example:

import nep
import time

node = nep.node("pub")
conf = node.conf_pub(network = "direct", ip = "192.168.11.1", port =  7070)
pub = node.new_pub("test", conf)

while True:
    msg = {"message":"hello world"}
    pub.send_info(msg)
    time.sleep(1)

and subscriber can be as follow:

import nep

node = nep.node("sub")
conf = node.conf_sub(network = "direct", ip = "192.168.11.1", port =  7070)
sub = node.new_sub("wizard", conf)
while True:
    s, msg = sub.listen_info()
    if s:
        print msg
