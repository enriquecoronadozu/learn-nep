Robot Interface from Zero Experience (RIZE)

We are still working in this documentation page

Last update 22/03/2018

RIZE (Robot Interface from Zero Experience) is Google Blockly, form and web-based programming interface which can be used to work with heterogeneous and multi-robot systems. RIZE was designed to enable an easy addition of new robots, sensory devices and human behavior understanding algorithms.

The main idea of RIZE is reduce the effort that requires robot development for both end-users and expert programmers using form- and block-based enviroments and automatic code generation tools.

Robot connection

In this interface users can connect a new robot. For each robot the users need to fill the following:

  • Robot type
  • Robot name: # For the moment not used
  • Robot IP # Ask me if you have problem with filling this field (if other set of number such as 127.0.0… is required), for the moment I only put the options that are present in the GVlab

The IP is defined in two parts:

  • IP (start): commun IP adress
  • IP (last): last number of the IP adress which generally changes when the netwrok fails or is restarted.

Example:

For a IP for nao robot of 192.168.0.11, you need to select 192.168.0 in the IP (start) dropdown input and write 11 in the IP (last) textbox input.

Code generation from the end-user point of view

Many end-users requires to program robots without learning a programming language or write a command line. RIZE provides an easy to get started method (block-based programming) to enable end-user to develop thier own applications.

Generally, a Human-Robot Interaction application follows the information processing cognitive model. This model basically separates the robot capabilities in the following processors:

  • Percpetion
  • Cognition
  • Action

Robot programming

The main RIZE interface for programming consist of a Toolbox (1) in which the user selects the programming blocks to use and a Workspace (2) where the user can drag and drop these blocks to generate a program. RIZE also provides some buttons in the Main menu (3) that supports some usable features such as save, load and recover block-based programs, see the generated code and see block documentation.

Program menu (5) enable to undo and redo a version of the program as well as run, stop, restart the programs.

The main idea of RIZE is to use a block-based environment, which are considered as easier to get started and more engaging for novice programming learners to enable interaction designers to launch the desired components (without knowing how to use using complex XML-based launchers or command line tools) and generate their own applications.

In RIZE, the code needed to define the behaviors using is automatic generated. This definition is performed by the end-user as a set of concatenated primitives. Robot actions can be defined using the block presented in figure 3. An action block can takes a simple primitive block or a list of primitives to execute parallel tasks. It can also take a list of robots to perform the defined action in multi-robots systems (in the future this will be defined by the robot name instead the robot type).
