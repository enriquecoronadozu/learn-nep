What is NEP (Node Primitives)?

A cross-platform robot programming framework for Robot-Human interaction.

We are still working in this documentation page

Last update 29/01/2018

Main features of NEP?

  • Easy to install
  • Open source
  • Full Windows, Linux and MacOSX support
  • Compatible with ROS (Robot Operating System)

What you can do with NEP?

  • Creates robot components using client-server and publish-subscribe communication patterns.
  • Design complex Human-Robot interaction applications
  • Communicate components written in different programming languages such as Python, C++ C#, Matlab, etc.

Test a simple publish-subscribe application

In order to test that NEP framework was suceffully installed we will create a simple publisher-subcriber application. For this we need to create 2 python files.

The first one will be a node with a publisher. This script can be denoted as pub.py. This script must to have the next code:

import nep
import time

node = nep.node("pub")
conf = node.conf_pub(network = "direct")
pub = node.new_pub("test", conf)
while True:
    msg = "hello world"
    pub.send_string(msg)
    time.sleep(1)

The second one will be a node with a subscriber. This script can be denoted as sub.py. This script must to have the next code:

import nep

node = nep.node("sub")
conf = node.conf_sub(network = "direct")
sub = node.new_sub("test", conf)
while True:
    s, msg = sub.listen_string()
    if s:
        print msg

Then, execute both scripts in different terminal/console windows. If the message of hello world appears in the console of the subscriber node, then the installation has been successful.

About tutorials and examples

The example before works in default port in local network. In order to create more complex robot applications, we need to use a more convenient network architecture. Please see the Basic tutorials page to learn more about distributed systems and to see more useful examples for robot programming.
