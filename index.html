Get started with NEP

Node primitives (NEP) is a front-end robot programming framework for Robot-Human Interaction

We are still working in this documentation page

Last update 22/03/2018

Main features of NEP?

NEP have been designed to be usable and interoperable.

Usability features:

  • Easy to install
  • Use of a human-centered API

Interoperability features:

  • Cross-platform (Windows, MacOS and Linux)
  • Use of high performance (ZeroMQand nanomsg) and widely used (ROS) middlewares as back-end
  • Use of JSON format for robot behavior specification

Other features:

  • Open Source
  • Mainly written in python
  • Component-based

What you can do with NEP?

  • Creates cross-platform robot components
  • Use client-server and publish-subscribe patterns to communicate these components
  • Use JSON format to specify complex robot behaviors based in the concatenation of primitive actions

Installation in Windows

Before installing NEP in Windows you need to install Python 2.7 for architecture x86. We recomend to use the Windows x86 MSI Installer from offical Python page.

https://www.python.org/downloads/release/python-2714/

Even when NEP can work with the Windows x86-64 MSI installer and other Python distributions such as Anaconda, the recommended version has better support for some robots, such as NAO and Pepper.

Option 1: NEP from installer

Click here to download experimental version

Follow the installation wizard (you need a intenet connection due that it will dowload some python packages). A black window will appears installing all the needed python packages (using pip) and building the NEP library. Please don’t close the black window until it close automatically.

Option 2: NEP From Github

Clone or download the most recent version of NEP from:

https://github.com/enriquecoronadozu/NEP

Then run the install.bat file (with double click).

Test a simple publish-subscribe application

In order to test that NEP framework was suceffully installed we will create a simple publisher-subcriber application. For this we need to create 2 python scripts.

The first one will be a node with a publisher. This script can be denoted as pub.py. This script must to have the next code:

import nep
import time

node = nep.node("pub")
conf = node.conf_pub(network = "direct")
pub = node.new_pub("test", conf)
while True:
    msg = "hello world"
    pub.send_string(msg)
    time.sleep(1)

The second one will be a node with a subscriber. This script can be denoted as sub.py. This script must to have the next code:

import nep

node = nep.node("sub")
conf = node.conf_sub(network = "direct")
sub = node.new_sub("test", conf)
while True:
    s, msg = sub.listen_string()
    if s:
        print msg

Then, execute both scripts in different terminal/console windows. The message of hello world need to appears in the console of the subscriber node.

About tutorials and examples

The example before works in default port in local network. In order to create more complex robot applications, we need to use a more convenient network architecture. Please see the Basic tutorials page to learn more about distributed systems and to see more useful examples for robot programming.
