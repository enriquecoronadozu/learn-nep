JSON Robot Behavior Specification (JRBS)

We are still working in this documentation page

Last update 16/04/2018

High level, close of mapping and modular robot behaviors are be defined in NEP/RIZE using Robot Behavior Specificiation Language based in JSON format. These behaviors can be simple atomic primitives, complex composite actions, conditions and other decision making definitions. The way how behaviors can be specified in this format is shown in this section.

Primitives

A simple primitive can be expressed as a python dictionary as shown bellow:

p = {"primitive":"say", "input":"hello", "options"{"velocity":50}}

A primitive is defined in a JSON (python dictionary) format using 3 main fields:

  • primitive: this field is used to define the name of the primitive. This field must be defined as a string value.
  • input: this field is used to specify the main input of this primitive. This field can have any type of value.
  • options: this field is used to specify more advanced configuation parameters that better define the primitive. This field must be defined as a python dictionary.
